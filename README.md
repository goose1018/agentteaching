# AI 名师分身 Web MVP

这是第一版网页版原型，目标是先把产品流程跑通：

- 官网端：商业化首页，解释产品定位、三方价值、信任机制和试点方案。
- 学生端：ChatGPT 式布局 + 微信式聊天气泡 + 图片上传入口。
- 老师端：AI 出题，老师语音讲解，系统抽取方法卡。
- 审核端：老师审核 AI 回答，形成正例和反例。
- 模型端：不同任务可切换 GPT、DeepSeek、Kimi、国内视觉模型或本地开源模型。

## 入口

- 商业官网：`#/home` 或默认打开。
- 学生体验：`#/student`。
- 老师训练端：`#/teacher`。

## 运行

```bash
npm install
npm run dev
```

## 当前状态

当前版本是前端 MVP，AI 回答是模拟的。后续接真实服务时建议按以下顺序：

1. 接图片上传和视觉模型读题。
2. 接老师语音转文字。
3. 把老师讲稿抽取成方法卡和示范问答。
4. 接 pgvector 或其他向量库做 RAG。
5. 接模型路由层，根据任务切 DeepSeek/Kimi/GPT/国内视觉模型。
6. 加用户登录、用量限制、支付和合规协议。

## 商业化注意

国内正式开放时，不建议把 GPT 作为唯一底层模型。更稳的路线是：

- 内部效果标杆：GPT。
- 老师演示：GPT + 国内模型双跑。
- 国内试点：DeepSeek V4 / Kimi / 通义等国内可用模型。
- 正式上线：走合规模型、内容安全、未成年人保护和备案/登记评估。
